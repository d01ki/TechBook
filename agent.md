# Agent.md

## プロジェクト概要
検索ワードを起点として、Zenn・Qiita・はてなブログ等の技術記事を**自動収集・一覧表示**するシンプルなWebアプリ。

- 興味 = 検索キーワード
- 学習・推薦ロジックは持たない
- 「探さなくていい」を最小構成で実現

一般ユーザーが直感的に使え、実装は可能な限り軽量に保つ。

---

## ゴール
- 検索するだけで複数メディアの記事を横断取得できる
- 新着技術記事を効率よく把握できる
- JavaScript（TypeScript）中心で完結する

---

## 非ゴール（やらないこと）
- ユーザー登録・ログイン
- 興味の自動学習・推薦アルゴリズム
- 記事本文の全文保存・転載
- 要約生成・高度なNLP

---

## 想定ユーザー
- 技術記事を読む習慣はあるが、巡回が面倒な人
- Qiita / Zenn を横断して情報収集したい人
- RSSリーダーは難しそうだと感じている人

---

## ユーザーフロー
1. トップページにアクセス
2. 検索欄にキーワードを入力（例: `JavaScript セキュリティ`）
3. 複数メディアから該当記事が一覧表示される
4. 記事をクリックすると元サイトへ遷移

---

## 機能要件（MVP）
### 必須
- 検索フォーム
- 記事取得（Zenn / Qiita / はてな）
- 記事一覧表示
- 外部リンク遷移

### あると良い（後回し可）
- ソース別フィルタ
- 新着順ソート
- 既読管理（localStorage）

---

## 技術構成
### フロントエンド
- Next.js (App Router)
- TypeScript
- Fetch API

### バックエンド（最小）
- Next.js API Routes / Server Actions
- Node.js

役割：
- RSS取得
- 記事データの正規化
- クライアントへJSON返却

---

## データ取得方針
### RSSベース
- Qiita: タグRSS / 全体RSS
- Zenn: トピックRSS
- はてなブログ: 技術カテゴリRSS

取得後、以下で簡易フィルタ：
- タイトルに検索キーワードを含むか

---

## データ構造
```ts
Article {
  title: string
  url: string
  source: 'Qiita' | 'Zenn' | 'Hatena'
  publishedAt: string
}
```

---

## API仕様（例）
### GET /api/search
**Query**
- q: string（検索キーワード）

**Response**
```json
{
  "articles": Article[]
}
```

## 運用・注意点
- 記事本文は保存しない
- サムネ・冒頭文のみ表示、必ず元リンクへ誘導
- 各サービスの利用規約を尊重
